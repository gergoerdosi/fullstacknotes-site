## üöÄ How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Clone** your fork locally:
    ```bash
    git clone [https://github.com/YOUR-USERNAME/fullstack-notes.git](https://github.com/YOUR-USERNAME/fullstack-notes.git)
    cd fullstack-notes
    ```
3.  **Install dependencies** (uses NPM/Astro):
    ```bash
    npm install
    ```
4.  **Create a branch** for your content:
    ```bash
    git checkout -b topic/new-react-note
    ```
5.  **Make your changes.** (See "Content Guidelines" below).
6.  **Test your changes** locally to ensure the site builds:
    ```bash
    npm run dev
    ```
7.  **Commit and Push**:
    ```bash
    git commit -m "Add note about React Hooks"
    git push origin topic/new-react-note
    ```
8.  **Open a Pull Request** against the `main` branch of this repository.

## üìù Content Guidelines

To keep "Fullstack Notes" high-quality, please follow these rules:

* **Be Concise:** These are notes, not long-form essays. Get straight to the point.
* **Code Blocks:** Always use syntax highlighting for code blocks (e.g., \`\`\`js).
* **Frontmatter:** Ensure every new `.md` or `.mdx` file has the required Starlight frontmatter (title, description).
* **No Plagiarism:** Do not copy-paste directly from other documentation sites (MDN, React Docs, etc.). Summarize in your own words.

## üêõ Found a bug?

If you find a typo or a factual error, feel free to open a PR directly! For larger structural changes, please open an Issue first to discuss it.
